  • Introducing iBot

    Interacting Bot is a movable robot that is not only capable of moving smartly by being controlled by a video game controller, but can also interact with you using its head(screen) while helping you understand the environment around you using Deep Learning techniques.The salient features of this bot are:

    • Controllable with the help of any Video Controller
    • Collision Avoidance with the help of array of ultrasound sensors attached to its base.
    • Automatic direction change in case of obstruction.
    • Capable of carrying considerable amount of weight.
    • Has the ability to scan QR code in order to access various information such as web links, images, text.
    • Uses Object Detection and Scene Understanding Framework in order to analyze its surrounding environment.
    • Capable of interacting with users using a virtual face.

Features

As briefly described above, iBot is a movable robot which can be controlled using a video game controller. However, if you are not comfortable in handling the robot or you are afraid that it might collide with other obstacles, No worries! We have got you covered. There are sensors present at right positions so whenever any other object comes in close radius of your robot, it will automatically change its direction towards a safer way without your intervention. Alternatively, if obstacles on all the sides cover it, then it will stop at its position until there is a clear path to proceed. (Hardware details explain)

With the help of a screen attached on the topmost platform of the robot, the bot is able to interact with its surroundings in different ways. There is a camera attached at the top that makes it capable of finding QR Codes in the scene. If there is any QR code present, the bot will speak about the contents. This functionality can be used if you want to use the bot as an automated guide for a museum or library or any such place. So if it is passing by any area and it happens to notice a QR code, it will stop and dictate its contents to the listener with an interactive talking face on the screen.

Apart from scanning QR code, the bot is also able to perform Scene Detection with Object Detection. With the help of scene detection framework, the bot is able to analyses the scene and is able to point out what major objects are visible with the help of object detection. It also shows how much sure it is about the prediction in form of percentage. Such kind of functionalities can be used for assisting physically disabled people who cannot see. This bot can help them on understanding their surroundings in a better way. As in the below give picture it specifies what object are present and also specifies their position by outlining them.

On more salient feature of this bot is that it can provide you constant company. Yes! You can interact with it, ask questions and it will provide you with best possible answers. This feature can be extended to use it as an assistant in helping you remember your appointments, important dates, reminders etc.

Design of the iBot:

We used a 3-layer structure for our moving Bot. Movement of the bot is facilitated by the use of three wheels, one in the front and two at rear. Rear wheels are controllable and are connected to DC Motors while the front wheel is freely moving; therefore, the bot is Rear-wheel drive. The bottom layer mostly holds the driving and decision making components of the bot. Major components on this level are motor driver, Six ultrasonic sensors, raspberry-pi. This Bot can be moved in all eight directions and is controlled using a wired video game controller. The mid layer is where the laptop resides and the user can keep their luggage. The top layer acts as the head of the bot where the LCD screen and webcam is attached. This allows the bot to take visual input and display the processed output.

Hardware used:

Ultrasound Sensors: In order to make the bot capable of avoiding obstructions and to steer it in the desired direction, an array of 6 ultrasound sensors(HC-SR04) has been used to measure the distance of various obstacles around the bot. The figure below shows the arrangement of the sensors that has been used to get distance in all the directions possible.

Motor driver: In order to make the bot capable of carrying considerable amount of weight and to make its motion smooth, motor driver (L298N) has been used. Use of motor driver is essential as a microcontroller has a very low output current that is not capable to drive directly more than an usual indicator LED with it. The motor draws a much higher current allowing the motors to work at higher rpm and thus its pay load ability.

LCD Screen and camera: To make the bot able to interact with the user and to allow it to capture the scenes, a LCD screen and a webcam has been attached to the bot using a mobile stand. This allows the bot to interact with the user and display the information.

Software/Processes used:

QR code detection:For the detection of QR codes from the live scene, we have used Zbar library. Whenever the bot comes across any QR code, it stops and performs the next set of instruction according to the QR code data. OpenCV facilitates the process of reading barcodes and QR codes, including loading an image from disk, grabbing a new frame from a video stream, and processing it. Once we have the image or frame we can then pass it to a dedicated Python barcode decoding library such as a Zbar. The ZBar library will then decode the barcode or QR code. OpenCV can come back in to perform any further processing and display the result. So here’s the game plan.

Our barcode detection in video system can be broken into two components:
Component #1: A module that handles detecting barcodes in images (or in this case, frames of a video).
Component #2: A driver program that obtains access to a video feed and runs the barcode detection module.

Speed: The ZBar library is almost twice as fast as the OpenCV QR code detector. Robustness: The ZBar library produces more robust results as compared to OpenCV on the following factors as shown in the above video, ZBar is better or comparable at various rotation, ZBar is better at different image sizes as seen from the different zoom levels in the video, ZBar is better at handling perspective distortions ( when the image is not upright w.r.t the camera).

Interactive Face: For the graphics of speaking emoji part, we have used Pygame library. The speaking emoji works in sync with the audio and gives a feeling of actually talking to someone. The picture above shows the face that has been using the above mentioned library.

Threading: We used Multithreading to execute QR code scanner and face of iBot in parallel. A thread is a sequence of such instructions within a program that can be executed independently of other code. Multithreading is the ability of a processor to execute multiple threads concurrently. Multiple threads can exist within one process where each thread contains its own register set and local variables and all thread of a process share global variables and the program code.

Scene Captioning (Microsoft Caption Bot): Our bot is capable of understanding the scene and describing it to the user. This mode can be enabled by two ways.

1- By directly showing a QR code containing a link of any online image file.
2- By using a particular QR code available with the bot to use its scene detection mode.

Once it comes across this code, it enters in the live imaging mode. In this mode you can click spacebar whenever you want to click an image of a live scene. This image will be saved and a description of its contents will be given by our interactive screen. For analyzing the images, we have used an AI API available from Microsoft.

Object detection: To allow the bot to be able to detect various objects present in the visual image that it captures using the webcam, we have used YOLOv3(You Only Look Once) framework. YOLO is the state of the art object detection framework that is the present fastest available algorithm available. However, due to limitation in the processing power available in the laptop being used, the code takes around 40 seconds per image.
